Labby design:

Advanced features implemented:

XSLT viewer produce lab reports in 2 predefined XSLT formats and ability to upload a user defined XSL and render it into the reports page. XSL file are rendered on the client side, thanks to javascript this load can be taken off the browser.
Instrument search engine all users have the ability to search for instruments (from the instruments table in the DB). The search is done with MySQL's MyISAM engine. Search prioritizes instruments according to the user permissions and keywords found, in order to enable users to search for new instruments (for future use), they have the ability to see instruments that they have no permission to use them.

Labby's architecture

Uniform look Labby was designed to be attractive yet simple to modify. It's easily scalable and adding new features is an easy job. Moreover, its designed so it would scale in different browsers and resolutions.
jQuery/UI Different parts of the website contain jQuery, helping us to write simple yet efficient javascript.
Smart UI Each page was written in order to enable the user to use the website without any hassle while trying to protect as much and expose as little of the system mechanics.
E-Mail recovery Password recovery for users that entered an email while signing up.
Sortable/Searchable Tables Large tables can be sorted by fields and some are searchable, large and long pages are always too complicated.
AJAX Editable pages are written using AJAX for faster and better form processing.
Reports Reports are generated directly from the DB into XML and Interactive charts, long reports are boring and hard to read, seeing them in a simple pie chart is more informative.
Server configuration Server parameters (DB name, user, password, etc.) are configured in web.xml and can be managed easily (no code recompile).
